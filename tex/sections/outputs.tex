\section{Outputs}
\label{sec:outputs}

\subsection{Overview of XML configuration files}

The content and frequency of the \NEMOSIcu{} output files are configured via the files \verb|file_def_nemo-oce.xml| and \verb|file_def_nemo-ice.xml| in the run directory.
The former controls ocean output, the latter sea ice output.
The files are written in XML, which is similar to HTML.
The full story is in Chapter~12 of the \citetalias{nemo-manual}; this section just covers the basic practicalities.

We will refer to the completed run of section~\ref{sec:tutor-running} as a starting point.
In that section, the template \verb|file_def_nemo-*.xml| files from the auxiliary resources replace the default ones.
The model outputs for that test run are thus deliberately limited: there are 12 different variables at two different frequencies and these come to about 54\,MB in total for the 1-year test run.
If one does the same simulation but using the original XML files to configure the outputs, the same run yields 156 output variables in total, coming to about 5.4\,\textbf{G}B.
It is clear how quickly (shared) storage space would be consumed with multiple, longer runs, and yet the majority of those variables are likely not needed.
This is the reason for the use of the lightweight template files in the first instance: it is better to start with a relatively `clean slate' and add variables as required rather than use the default bloated lists and decide what to remove or keep.

That said, those templates are probably \textit{too} limited for typical usage, so this section describes how to add additional fields using those XML files as a base.

Firstly, note the following general structure and syntax of the XML files:

\begin{verbatim}
  <file_definition ... >
     <file_group id="group_id_name" output_freq="5d" ... >
        <file id="file_id_name" ... >
           <field field_ref="some_variable_name" ... />
        </file>
     </file_group>
  </file_definition>
\end{verbatim}

where \verb|...| are additional \verb|name="value"| attribute pairs, separated by spaces.
At the inner-most level is a list of \verb|<field>| tags (just one is shown above), which identifies (and optionally configures) the output variables to go into the file with the \verb|<file>| tags surrounding them.
Multiple \verb|<file>| tags can be defined (again just one is shown above) within a surrounding \verb|<file_group>| tag, the latter specifying common attributes (notably, the output frequency) to all of the \verb|<file>|s it contains.
Again, more than one \verb|<file_group>| can be defined in general, and they are all contained within an outer-most \verb|<file_definition>| tag which sets at a more technical level how the files are to be written.
Each tag is described below, starting with the inner-most tags and working outwards.

\subsection{File and field tags}
\label{sec:outputs-field-tag}

The output file:
\begin{verbatim}
  ORCA2_5d_00010101_00011231_icemod_0001-0001.nc
\end{verbatim}

contains 5-day means of a few basic sea ice variables, as configured in\linebreak\verb|file_def_nemo-ice.xml| by these lines:

\begingroup\small
\begin{verbatim}
  <file id="file101" name_suffix="_icemod" description="sea ice variables">
     <field field_ref="iceconc"      name="siconc"    />
     <field field_ref="icevolu"      name="sivolu"    />
     <field field_ref="icethic"      name="sithic"    />
  </file>
\end{verbatim}
\endgroup

\textbf{Define additional output variables by adding more \texttt{<field>} tags between the \texttt{<file>...</file>} lines.}
The file \verb|field_def_nemo-ice.xml|, which should be linked in your experiment directory, catalogues all the available sea ice outputs, and there is similarly a \verb|field_def_nemo-oce.xml| for ocean variables.
Refer to these files to identify the variables you want to output.
When adding a new variable to your \verb|file_def_nemo-ice.xml|, the \verb|field_ref| attribute is set to the \verb|id| attribute of the corresponding variable in \verb|field_def_nemo-ice.xml|.
The optional \verb|name| attribute in each \verb|<field>| line is the name that the variable will have in the netCDF file.

You can define new files by adding a new \verb|<file>...</file>| structure within a file group (see section~\ref{sec:outputs-group-tag}).
The \verb|id| attributes of all the \verb|<file>| tags (e.g., see first line above where it is set to \verb|"file101"|) must be unique---including files defined in \verb|file_def_nemo-oce.xml|.
They should have the form \verb|"fileN"| where \verb|N| is an integer between 1 and 999.
For example, the template \verb|file_def_nemo-ice.xml| has an additional file with \verb|id="file102"| for scalar quantities.
This separation is logical but it is not required that all variables in a file have the same spatial dimensions.
However, all variables in a file will always have the same time dimension, as this is set at the \verb|<file_group>| level (see section~\ref{sec:outputs-group-tag}).

Other optional attributes in the \verb|<file>| tag definition are \verb|name_suffix|, which is inserted into the file name (see section~\ref{sec:outputs-file-tag}), and \verb|description|, which goes into the netCDF metadata.


\subsection{File group tag}
\label{sec:outputs-group-tag}

Output files are grouped within \verb|<file_group>| tags, which serve two key purposes.
Foremost, they declare the output frequency for each of the \verb|<file>|s it contains.
The \verb|<file_group>| for the sea ice outputs described in section~\ref{sec:outputs-field-tag} is as follows:

\begingroup\footnotesize
\begin{verbatim}
  <file_group id="ice_5d" output_freq="5d" split_freq="1y" enabled=".TRUE.">
     <!-- file101 -->
     <!-- file102 -->
  </file_group>
\end{verbatim}
\endgroup

The \verb|id| attribute of the \verb|<file_group>| tag is required and must be unique.
The attribute \verb|output_freq| is, obviously, the output frequency of the variables in each of the files within the group.
Here, it is set to 5-daily.
You can change it to, for example, daily by replacing the \verb|"5d"| with \verb|"1d"|, or monthly by changing it to \verb|"1mo"|.
In both cases it would make sense to also update the \verb|id| attribute, although this is not necessary beyond the \verb|id| having to be unique.
In the template \verb|file_def_nemo-ice.xml|, a second \verb|<file_group>| is declared for monthly outputs, which contains two \verb|<file>|s having the same contents as the corresponding 5-daily files.
Note that those two files have different \verb|id| attributes (\verb|file103| and \verb|file104|).

The \verb|split_freq| attribute defines how often output files are divided up in time.
Here, it is set to \verb|"1y"|, meaning that a new file will be started every year of simulation, which is a good typical default (see the outputs of section~\ref{sec:inputs-core-ciaf} for an example).
It can be changed to, for example, \verb|"1mo"| to split every month, or \verb|"1d"| for daily files, perhaps useful if you have larger output files.

The \verb|enabled| attribute can either be set to \verb|".TRUE."| or \verb|".FALSE."| (case insensitive) or omitted entirely.
If you change it to \verb|.FALSE.| and then run the simulation, you will find the files within the group are no longer output.
This attribute can also be added to each \verb|<file>|, which then overrides the corresponding state in its \verb|<file_group>|, allowing one to individually enable or disable file outputs.
When the attribute is defined in both cases, note it is inherited `inwards' with priority taken by the value in the \verb|<file>| tag:
\begin{center}
    \begin{tabular}{ccc}
        \multicolumn{2}{c}{\texttt{enabled} in tag:} & File is output? \\
        \verb|<file_group>| & \verb|<file>|          &                 \\
        \hline
        \verb|".TRUE."|     & Not set                & Yes             \\
        \verb|".FALSE."|    & Not set                & No              \\
        \verb|".TRUE."|     & \verb|".TRUE."|        & Yes             \\
        \verb|".TRUE."|     & \verb|".FALSE."|       & No              \\
        \verb|".FALSE."|    & \verb|".TRUE."|        & Yes             \\
        \verb|".FALSE."|    & \verb|".FALSE."|       & No
    \end{tabular}
\end{center}



\subsection{File definition tag}
\label{sec:outputs-file-tag}

The outer \verb|<file_definition>| tag contains some pre-defined attributes that generally don't need to be changed, except the \verb|name| attribute which sets the file name format.
The default uses some macros (surrounded by @) that extract information about the run from within NEMO and/or the inner \verb|<file_group>| tags.
There are two useful changes that can be made regarding the \verb|name| attribute:

\begin{itemize}
    \item Change the namelist parameter \verb|cn_exp| in namelist group \verb|&namrun| of \verb|namelist_cfg|; this will then substitute the \verb|@expname@| macro in the \verb|name| attribute.
    \item Prepend a path to the name to change the output directory to something else, e.g.:

\begingroup\small
\begin{verbatim}
   name="/path/to/somewhere/@expname@_@freq@_@startdate@_@enddate@"
\end{verbatim}
\endgroup

\end{itemize}
